오늘 하루 동안 모든 Claude Code 세션에서 작업한 내용을 수집해서 정리하고 GitHub에 저장해줘.

인자가 있으면 (예: `/퇴근 어제`, `/퇴근 2025-03-04`) 해당 날짜 기준으로 작성해.

## 0단계: 레포 준비 & 사용자 확인
```bash
REPORT_DIR="$HOME/work-reports"
if [ ! -d "$REPORT_DIR/.git" ]; then
  git clone https://github.com/rircom/work-reports.git "$REPORT_DIR"
fi
git -C "$REPORT_DIR" config user.name
```
이 값을 {username}으로 사용.

## 1단계: 날짜 확인
```bash
date +"%Y-%m-%d %A %u"
date -d "tomorrow" +"%d"
```
인자가 "어제"면 `date -d "yesterday"`, 날짜 문자열이면 해당 날짜 사용.

## 2단계: 모든 세션 작업 수집
```bash
python3 "$HOME/work-reports/scripts/collect-today.py" [날짜인자]
```
이 스크립트가 수집하는 것:
- **모든 프로젝트의 오늘 git 커밋** (어떤 세션에서 했든 전부)
- **오늘의 모든 Claude 세션 대화 내용** (모든 프로젝트, 모든 창)

이 수집 결과를 기반으로 작업일지를 작성해.

## 3단계: 최신 pull
```bash
cd "$HOME/work-reports" && git pull --rebase 2>/dev/null || true
```

## 4단계: 일지 작성
파일 경로: `$HOME/work-reports/daily/YYYY-MM-DD-{username}.md`

수집된 데이터를 종합 분석해서 아래 형식으로 작성:
```markdown
# 작업일지 YYYY-MM-DD ({username})

## 오늘 한 일
- (모든 세션에서 작업한 내용을 프로젝트별로 정리)

## 주요 변경사항 (Git)
- (프로젝트별 커밋 요약)

## 해결한 문제
- (해결한 문제나 버그)

## 만든 것 / 배운 것
- (새로 만든 파일, 기능, 알게 된 것)

## 다음에 이어서 할 일
- (미완성이거나 이어서 해야 할 작업)

## 메모
- (기타 기억해둘 것)
```

## 5단계: 주간 요약 (금요일에만)
요일이 금요일(5)이면 `$HOME/work-reports/weekly/YYYY-WNN-{username}.md` 파일도 저장.
이번 주 daily 파일들을 읽어서 주간 핵심만 요약.

## 6단계: 월간 요약 (말일에만)
오늘이 월의 마지막 날이면 `$HOME/work-reports/monthly/YYYY-MM-{username}.md` 파일도 저장.

## 7단계: Git commit & push
```bash
cd "$HOME/work-reports"
git add daily/ weekly/ monthly/
git commit -m "$(cat <<'EOF'
작업일지: YYYY-MM-DD ({username})

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
EOF
)"
git push
```

## 8단계: 결과 보고
저장한 파일 목록과 GitHub URL을 알려주고, "퇴근 완료!" 로 마무리.
