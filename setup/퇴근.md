오늘 작업한 내용을 정리하고 GitHub에 저장해줘. 아래 절차를 순서대로 따라해.

인자가 있으면 (예: `/퇴근 어제`, `/퇴근 2025-03-04`) 해당 날짜 기준으로 작성해.

## 0단계: 사용자 확인
```bash
git -C /mnt/h/Claude/work-reports config user.name
```
이 값을 {username}으로 사용.

## 1단계: 날짜 확인
```bash
date +"%Y-%m-%d %A %u"  # %u = 1(월)~7(일), 금요일=5
date -d "tomorrow" +"%d"  # 내일이 1이면 오늘이 말일
```
인자가 "어제"면 `date -d "yesterday"`를, 날짜 문자열이면 해당 날짜를 사용.

## 2단계: 최신 pull
```bash
cd /mnt/h/Claude/work-reports && git pull --rebase 2>/dev/null || true
```

## 3단계: 오늘 일지 저장 (매일)
파일 경로: `/mnt/h/Claude/work-reports/daily/YYYY-MM-DD-{username}.md`

아래 형식으로 작성:
```markdown
# 작업일지 YYYY-MM-DD ({username})

## 오늘 한 일
- (오늘 대화에서 작업한 내용 핵심 요점)

## 해결한 문제
- (해결한 문제나 버그)

## 만든 것 / 배운 것
- (새로 만든 파일, 기능, 알게 된 것)

## 다음에 이어서 할 일
- (미완성이거나 이어서 해야 할 작업)

## 메모
- (기타 기억해둘 것)
```

## 4단계: 주간 요약 (금요일에만)
요일이 금요일(5)이면 `/mnt/h/Claude/work-reports/weekly/YYYY-WNN-{username}.md` 파일도 저장.
이번 주 daily 파일들을 읽어서 주간 핵심만 요약.

## 5단계: 월간 요약 (말일에만)
오늘이 월의 마지막 날이면 `/mnt/h/Claude/work-reports/monthly/YYYY-MM-{username}.md` 파일도 저장.
이번 달 daily 파일들을 읽어서 월간 성과/주요 작업 요약.

## 6단계: Git commit & push
```bash
cd /mnt/h/Claude/work-reports
git add daily/ weekly/ monthly/
git commit -m "$(cat <<'EOF'
작업일지: YYYY-MM-DD ({username})

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
EOF
)"
git push
```

## 7단계: 결과 보고
저장한 파일 목록과 GitHub URL을 알려주고, "퇴근 완료!" 로 마무리해줘.
GitHub URL: https://github.com/rircom/work-reports/blob/main/daily/YYYY-MM-DD-{username}.md
